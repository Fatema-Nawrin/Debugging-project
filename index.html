Javascript Questions & Answers

How does Javascript work?

Every browser has Javascript engines to run javascript code. Chrome has V8 engine which has a parser that creates a syntax tree. Then, ignition interpreter generates bytecode from this syntax tree. There is a JUST IN TIME compiler called TurboFan. It creates optimized machine code from the bytecode. Basically, to run javascript, it needs to turn the human readable code into machine code.

What does event loop do?

Javascript is single threaded, non-blocking & asynchronous. Javascript has 3 sections for execution of code, call stack, web API & task queue. Call stack is a data structure where code is executed. If there is a web API function, call stack send it over to web API. After Web API finishes executing the task, it goes to the Task Queue.

If the call stack is empty, first thing from task queue is pushed into the stack. After that wait for the stack to be cleared again & repeat the process until task queue is empty. This process is called event loop.

What are the differences between Local Storage vs Session Storage?

The main differences between local storage and session storage is that data in session storage is cleared when the tab is closed and session ends but data in local storage doesn't expire unless it is deleted manually. Local storage data is accessible from any tab but session storage data has to be accessed from the same tab. Session storage has 5mb capacity and local storage has 5/10mb capacity.
